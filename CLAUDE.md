## Architecture

This is a **React Native Turbo Module** library scaffolded with `react-native-builder-bob`. It wraps libyuv (Google's YUV conversion/scaling C++ library) for use in React Native apps.

### JS/TS Layer (`src/`)

- `NativeLibyuvResizer.ts` ŌĆö Turbo Module spec; defines the native bridge interface via `TurboModuleRegistry`
- `multiply.native.tsx` ŌĆö native implementation entry (calls `NativeLibyuvResizer`)
- `multiply.tsx` ŌĆö JS/web fallback
- `index.tsx` ŌĆö public exports

The codegen config in `package.json` (`codegenConfig`) generates the native spec glue from `NativeLibyuvResizer.ts`. Module name is `LibyuvResizer`, codegen name is `LibyuvResizerSpec`.

### Android (`android/`)

- `LibyuvResizerModule.kt` ŌĆö extends `NativeLibyuvResizerSpec` (codegen-generated), implements native methods
- `LibyuvResizerPackage.kt` ŌĆö registers the module as a Turbo Module via `BaseReactPackage`
- `build.gradle` ŌĆö no C++ build configured yet; add `externalNativeBuild` + CMake when adding libyuv

**libyuv integration (planned):** libyuv is a git submodule at `libyuv/` sourced from `https://android.googlesource.com/platform/external/libyuv/`. Android CMake uses `add_subdirectory(../libyuv libyuv)` and links against the `yuv` static target. iOS does NOT use libyuv ŌĆö use Apple's Accelerate framework (`vImage`) instead.

### iOS (`ios/`)

- `LibyuvResizer.mm` ŌĆö Objective-C++ implementation; extends the codegen-generated `NativeLibyuvResizerSpecJSI`
- `LibyuvResizer.h` ŌĆö header

### Example App (`example/`)

Separate yarn workspace (`react-native-libyuv-resizer-example`). Run example commands via `yarn example <cmd>` from root or directly inside `example/`.

## Key Details

- **Package manager:** Yarn 4 (Berry) with workspaces ŌĆö always use `yarn`, never `npm`
- **Build output:** `lib/` ŌĆö generated by `react-native-builder-bob` targeting ESM module + TypeScript declarations
- **Commits:** conventional commits enforced via commitlint + lefthook (`feat:`, `fix:`, `chore:`, etc.)
- **New Arch only:** module uses Turbo Modules (`isTurboModule = true`) ŌĆö no legacy bridge support
