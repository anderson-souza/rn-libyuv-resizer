# Architecture

**Pattern:** React Native Turbo Module library (New Architecture only)

## High-Level Structure

```
JS/TS (src/)
    │
    ├── NativeLibyuvResizer.ts  ← TurboModule spec (codegen source-of-truth)
    │       ↓ (codegen at build time)
    ├── multiply.native.tsx     ← calls NativeLibyuvResizer directly
    └── multiply.tsx            ← web/non-native fallback (throws)
                │
                ↓  (JSI bridge — zero-copy, no serialization)
        ┌───────────────────────────────────────┐
        │  Android                              │
        │  LibyuvResizerModule.kt               │
        │    extends NativeLibyuvResizerSpec     │
        │    → (planned) JNI → C++ libyuv       │
        └───────────────────────────────────────┘
        ┌───────────────────────────────────────┐
        │  iOS                                  │
        │  LibyuvResizer.mm                     │
        │    conforms NativeLibyuvResizerSpec    │
        │    → (planned) Apple Accelerate vImage │
        └───────────────────────────────────────┘
```

## Identified Patterns

### Turbo Module Spec Pattern

**Location:** `src/NativeLibyuvResizer.ts`  
**Purpose:** Single source of truth for the native API contract; React Native codegen generates C++/Kotlin/ObjC++ bridge stubs from this file.  
**Implementation:** `TurboModuleRegistry.getEnforcing<Spec>('LibyuvResizer')` — throws at runtime if native module not found (enforcing, not optional).

### Platform Split via `.native.tsx`

**Location:** `src/multiply.native.tsx` vs `src/multiply.tsx`  
**Purpose:** Metro bundler resolves `.native.*` for device, plain `.*` for web/Jest.  
**Implementation:** Native file calls `NativeLibyuvResizer`; web file throws a clear "native-only" error.

### Package Registration (Android)

**Location:** `android/LibyuvResizerPackage.kt`  
**Purpose:** Registers `LibyuvResizerModule` with React Native's module system.  
**Implementation:** Extends `BaseReactPackage`, implements `getReactModuleInfoProvider` returning `isTurboModule = true`. No legacy bridge registration.

### iOS JSI Glue

**Location:** `ios/LibyuvResizer.mm`  
**Purpose:** Bridges Objective-C++ to the JSI C++ spec generated by codegen.  
**Implementation:** Returns `std::make_shared<NativeLibyuvResizerSpecJSI>(params)` from `getTurboModule:`.

## Data Flow

### Method Call (current — placeholder `multiply`)

```
JS: multiply(a, b)
  → JSI (synchronous, no serialization)
  → Android: LibyuvResizerModule.multiply(Double, Double): Double
  → iOS: [LibyuvResizer multiply:(double)a b:(double)b]: NSNumber*
  → returns synchronously to JS
```

### Planned: Resize/Rotate (async)

```
JS: resizeImage(uri, width, height) → Promise<string>
  → JSI
  → Android: decode JPEG → C++ libyuv scale → encode → resolve promise
  → iOS: decode → vImage scale → encode → resolve promise
```

## Code Organization

**Approach:** Flat, platform-split by extension  
**JS/TS:** `src/` — spec, platform impls, barrel export  
**Android:** `android/src/main/java/com/libyuvresizer/` — Kotlin module + package  
**iOS:** `ios/` — `.h` + `.mm` pair  
**C++ (planned):** `android/src/main/cpp/` or shared `cpp/` at root (mirroring vision-camera-resize-plugin)  
**Example app:** `example/` workspace — isolated RN app for manual testing
